# プロジェクト議事録：個人用iPhoneアプリ開発と自宅LLM連携

**日付:** 2026-02-19
**目的:** AI（Claude Code/Gemini）を活用した個人用iPhoneアプリ開発手法の確立および、自宅マシンのLLMリソース活用基盤の構築。

## 1. 検討の経緯
### 初期フェーズ：開発ツールの選定
- **課題:** プログラミング知識なしでiPhoneアプリを開発したい。
- **検討:** 「Claude Code（ターミナル操作）」と「Web版LLM（Claude 3.7 / Gemini 1.5 Pro）」の比較。
- **決定:** - ターミナル操作のハードルを考慮し、**「Web版LLM（Google AI Studio等）でコード生成 → Xcodeへコピペ」**の手法を採用。
  - 開発アカウント（Apple Developer Program）は購入せず、無料の「Personal Team」署名（7日間有効・更新可能）運用とする。

### アイデア出し・ユースケース検討
- **方向性:** 市販アプリにはない「自分専用（n=1）」の課題解決を目指す。
- **決定したユースケース例:**
  - 広告なし・課金なしのシンプルツール（日記、瞑想タイマー等）。
  - APIを活用したWebサービス連携（Notion, Slack等）。
  - **【本採用】** 自宅のハイスペックPC（Mac）で稼働するLLMを外出先から操作するクライアントアプリ。

## 2. 技術スタックの策定
### アプリ開発基盤
- **言語:** Swift
- **UIフレームワーク:** SwiftUI（AIによるコード生成精度が高く、宣言的で記述量が少ないため）
- **開発環境:** Xcode 16+
- **データ保存:** `@AppStorage`（簡易データ）または SwiftData（構造化データ）。

### リモートLLM接続アーキテクチャ
- **サーバー側:** 自宅Mac + LM Studio (Local Serverモード)
- **ネットワーク:** Tailscale（VPN）を利用し、ポート開放なしで安全に接続。
- **クライアント側:** iPhone（Safariでの疎通確認 → SwiftUI製自作アプリへ移行）。

## 3. 現在のステータスとネクストアクション
- **ステータス:** アーキテクチャ設計完了。
- **直近のアクション:** アプリ開発の前に、既存ツールを用いた**「iPhone → 自宅Mac」の疎通確認（Connectivity Test）」**を実施する。
