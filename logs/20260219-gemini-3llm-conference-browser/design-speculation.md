# 設計仕様書：複数LLM会議ブラウザ

## 1. システム概要
1つのウィンドウ内で3つの異なるLLMモデルと同時並行で対話を行い、多角的な視点からの情報収集や議論（ブレインストーミング）を可能にするデスクトップアプリケーション。

## 2. 技術スタック
* **フロントエンド**: React, HTML/CSS (画面構築、UIコンポーネント)
* **バックエンド (アプリ基盤)**: Electron, Node.js (OS連携、ファイルシステム操作)
* **状態管理**: Zustand または React Context (会話ログの全体共有)
* **外部API**: OpenAI API, Anthropic API, Google Gemini API等

## 3. 画面レイアウト構成 (4ペインGrid配置)
ウィンドウを横に4分割（タブ1〜4）する。

* **タブ1 (司令塔/コントロールパネル)**
  * 全体一斉送信用の入力欄（入力欄1）および送信ボタン
  * クロス送信のON/OFFトグルスイッチ
  * 個別送信用の入力欄（入力欄2〜4）および送信ボタン
  * 「議事録を出力」ボタン
  * 「設定」画面へのアクセスボタン
* **タブ2〜4 (LLMチャットUI)**
  * 各LLM（例：タブ2=LLM A, タブ3=LLM B, タブ4=LLM C）との会話履歴を表示する吹き出し形式のUI。

## 4. 主要機能要件

### 4.1. ルーティング送信機能
* ユーザーの入力欄の選択に応じて、対象となるLLMのAPIへリクエストを振り分ける。

### 4.2. クロス送信機能
* トグルがONの状態で「入力欄1（一斉送信）」から送信された場合、直前のターンの他LLMの応答ログを取得。
* テンプレートに従い、「参考情報：他のAIの意見」としてユーザーのプロンプトに自動結合して各APIへ送信する。

### 4.3. 議事録エクスポート機能
* アプリのメモリ上に配列として保持している「全対話ログ（発言者、タイムスタンプ、内容）」を結合。
* システムプロンプト（要約指示）と共に設定されたLLM APIにリクエストを送信。
* 取得した要約テキストをプレビュー表示し、Electronの `dialog.showSaveDialog` 等を用いてローカルに `.md` または `.txt` 形式でファイル保存する。

### 4.4. ユーザー設定機能 (Settings)
* モデル選択機能：タブ2〜4、および議事録生成用にそれぞれ使用するLLMモデルをドロップダウン等で選択可能にする。
* APIキー管理：各ベンダーのAPIキーを入力・保存する。
* セキュリティ：保存されたAPIキーは平文で扱わず、`keytar` 等のライブラリを用いてOSのセキュアストレージに格納する。
